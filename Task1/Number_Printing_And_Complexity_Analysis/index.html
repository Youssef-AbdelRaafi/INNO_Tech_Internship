Number Printing Methods

Explore different approaches to print numbers from 1 to n without trailing comma

Test Input

Method 1: Loop with Condition Check Time: O(n) | Space: O(n)
function printNumbers1(n) { let result = ""; for (let i = 1; i <= n; i++) { if (i === n) { result += i; // Last number, no comma } else { result += i + ", "; // Add comma for all except last } } return result; }
How it works: Check if current number is the last one. If yes, don't add comma.

Pros

  • Simple logic
  • Easy to understand
  • Direct approach

Cons

  • Condition check in every iteration
  • String concatenation overhead
  • Not the most efficient

Method 2: Array Join Time: O(n) | Space: O(n)
function printNumbers2(n) { const numbers = []; for (let i = 1; i <= n; i++) { numbers.push(i); } return numbers.join(", "); // Join with comma separator }
How it works: Store all numbers in an array, then use join() to combine them with comma separator.

Pros

  • Clean and readable
  • Built-in join() is optimized
  • No manual comma handling

Cons

  • Extra memory for array
  • Two passes (fill + join)
  • Array allocation overhead

Method 3: String Builder Pattern Time: O(n) | Space: O(n)
function printNumbers3(n) { const parts = []; // Acts as StringBuilder for (let i = 1; i <= n; i++) { parts.push(i); } return parts.join(", "); } // Alternative implementation: function printNumbers3Alt(n) { let result = ""; if (n > 0) result += "1"; // Add first number for (let i = 2; i <= n; i++) { result += ", " + i; // Add comma + number for rest } return result; }
String Builder Pattern: In languages like Java/C#, StringBuilder optimizes string concatenation by using an internal buffer. JavaScript arrays with join() provide similar optimization.

Origin: String Builder pattern comes from the need to efficiently build strings without creating many intermediate string objects. It's built into languages like Java (StringBuilder), C# (StringBuilder), Python (list + join).

Pros

  • Optimized for string building
  • Avoids intermediate strings
  • Scalable performance

Cons

  • Slightly more complex
  • Memory overhead for buffer
  • Overkill for small strings

Method 4: Build then Remove Last Comma Time: O(n) | Space: O(n)
function printNumbers4(n) { let result = ""; for (let i = 1; i <= n; i++) { result += i + ", "; // Always add comma } // Remove last 2 characters (", ") return result.slice(0, -2); } // Alternative with substring: function printNumbers4Alt(n) { let result = ""; for (let i = 1; i <= n; i++) { result += i + ", "; } return result.substring(0, result.length - 2); }
How it works: Build the string with comma after every number, then remove the last comma and space.

slice() vs substring():
  • slice(-2) removes last 2 characters
  • substring(0, length-2) removes last 2 characters
  • slice() handles negative indices better

    • Pros

    • No condition in loop
    • Simple loop logic
    • Works for edge cases

    Cons

    • Extra string operation
    • Builds unnecessary characters
    • String concatenation overhead
